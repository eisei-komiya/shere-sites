キーボードの話

自作キーボードの世界

  • 自作キーボードとは、パーツを選んで組み立てるキーボード
  • キースイッチ(押し心地、キー音が変化)、キーキャップ、ケース、基板を自由に選択
  • ファームウェアでキー配列を完全にカスタマイズ可能

キーボード本体にキーマップを書き込むメリット

ソフトウェアでのキーマップ変更の課題

  • Karabiner Elements（Mac）やPowerToys（Windows）等のソフトがあるが...
  • 複数PC（私用・仕事用）で設定の同期が面倒
  • 複数OS（Mac / Windows / Linux）で別々に設定が必要
  • 会社によってはソフトのインストールが禁止されている場合も
解決策: キーボード本体にキーマップを書き込めば、どのPCに繋いでも同じ配列で使える
特別なソフトのインストールは一切不要

OS別レイヤーで違いを吸収

OSごとのショートカットの違い問題

操作 Mac Windows
アプリ切り替え Cmd + Tab Alt + Tab
行頭に移動 Ctrl + A Home

解決方法: OS別レイヤー

  • MacレイヤーとWinレイヤーを作成
  • 同じ位置のキーに、それぞれのOSのショートカットを割り当て
  • 接続先に応じてデフォルトレイヤーを切り替え
→ OSを意識せず、同じボタンで同じ動作を実現！

レイヤーという考え方

  • レイヤー = キーの役割を切り替える「層」
  • 特定のキーを押している間だけ、別のキー配列に切り替わる
  • 少ないキー数でも多くの機能を実現できる

例：40キーでも困らない理由

レイヤー 用途
Layer 0 通常の文字入力（QWERTY等）
Layer 1 数字・記号（1234...、!@#$...）
Layer 2 矢印キー・ナビゲーション
Layer 3 ファンクションキー・メディア操作

配列を変えられることの良さ

  • ホームポジションから手を動かさない
    • 矢印キー、マウス操作もホームポジションで
  • 自分だけの最適解を追求できる
  • よく使うキーを打ちやすい位置に配置
  • レイヤーで1つのキーに複数の役割
  • 小指の負担軽減（Ctrl、Shift等の配置変更）

カラムスタッガードとは

ロウスタッガード（通常）
横方向にずれている
タイプライターのアーム（活字棒）が干渉しないための設計の名残
→
カラムスタッガード
縦の列が指の長さに合わせてずれている
指の自然な動きに沿った配置
慣れるまでの壁: XとCを無限に打ち間違える日々...
でも1週間で慣れます。逆に広く普及してる普通の配列には戻れなくなるので一長一短。

半年使ってみた感想

  • bluetoothの接続先をワンタップで切り替えられるのが思いの外便利
    • macでアラートが出たので見つつ問題なかったらwindows/linux PCに戻るとか
  • 現状人気なトラボ付き分割無線は個人販売が主なため購入難易度高め
  • 高度なマクロを組もうと思ったら設定ファイルを自分でカキカキする必要がある
    • だが、大体AIで解決する(うごきゃいいし)
  • 慣れるまで大変だと思われがちだが、自分が一番自然だと思う配置にすればいいので実は認知負荷は低くすぐ慣れる
  • 自分が考える最強の配列への道は遠い。
    • QWERTY配列自体からの脱却はまだできてない(怖い)
  • OSレベルのショートカットを全く覚えなくていいのが最高
    • PCのリプレースや環境移行時にOS側はデフォルトショートカット設定で良いので楽
    総評: 自分の思い通りの動きをできる/させられるキーボード。カスタマイズ好きにはおすすめ。(vimとかvscodeの設定凝ってる人とか)
